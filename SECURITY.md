# 安全政策

## 报告安全漏洞

如果你发现安全漏洞，**请勿通过公开 Issue 报告**。

请通过以下方式私下报告：

1. 发送邮件至 [tbtparent@163.com](mailto:tbtparent@163.com)
2. 在 GitHub 中使用 Security Advisory 功能

请在邮件中包含：

- 漏洞描述
- 受影响的版本
- 复现步骤（如可能）
- 潜在影响

## 响应时间

- 我们将在 48 小时内确认收到报告
- 我们会尽快调查并修复漏洞
- 修复发布后，我们会进行安全公开披露

## 安全实践

本项目遵循以下安全实践：

- 依赖项定期更新
- 使用 npm audit 检查依赖安全
- 代码审查时关注安全问题
- 避免在代码中存储敏感信息
