# 数据查询安全（脱敏）

## 功能说明

接口响应中的敏感字段（手机号、邮箱、身份证、银行卡、姓名等）支持自动脱敏，无需修改业务代码。

脱敏规则在「系统管理 → 敏感字段配置」中维护，配置后立即生效。

---

## 支持的脱敏类型

| 类型 | 原始值 | 脱敏结果 |
|------|--------|----------|
| 手机号 | `13812345678` | `138****5678` |
| 邮箱 | `test@example.com` | `t***@example.com` |
| 身份证 | `110101199001011234` | `110***********1234` |
| 银行卡 | `6222021234567890123` | `**** **** **** 0123` |
| 姓名 | `张三` | `张*` |

---

## 配置方式

进入「系统管理 → 敏感字段配置」，新建配置项：

- **表名**：需要脱敏的数据库表名
- **字段名**：需要脱敏的字段
- **脱敏类型**：选择对应的脱敏规则
- **是否启用**：可随时开关，不影响原始数据

---

## 技术实现

脱敏通过响应中间件在数据返回前统一处理，原始数据不受影响。管理员角色可在系统配置中设置豁免，查看完整数据。
