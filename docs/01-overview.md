# Owl Platform 项目总览

## 背景与动机

在日常项目开发中，每一个面向 C 端的业务系统背后，往往都需要一套对应的管理后台：运营人员需要管理数据、开发人员需要查看日志、业务方需要实时看板……这些需求高度相似，却在每个项目中被反复从零搭建。

市面上现有的通用后台框架（如 vue-element-admin、AntD Pro 等）虽然提供了基础的 RBAC 权限体系和 UI 组件，但在实际落地时往往面临以下困境：

- **技术栈绑定问题**：大多数方案基于 Vue2/Vue3 或 React + 特定 UI 库，难以灵活切换
- **功能深度不够**：提供的是 UI 框架而非业务框架，日志、监控、文件管理、消息通知等能力需要自行实现
- **定制代价高**：贴近业务的部分（如数据权限、接口鉴权、代码生成）往往缺失或过于复杂
- **前后端割裂**：前端框架与后端完全解耦，从零接入成本高

基于以上痛点，Owl Platform 应运而生。

---

## 定位

Owl Platform 是一个**面向业务系统的通用管理后台基础框架**，目标是：

> 新建一个项目时，管理后台的基础能力开箱即用，开发者只需专注在业务逻辑本身。

它不是一个纯前端模板，而是一个**前后端一体的完整框架**，包含从数据库到页面的全链路实现。

---

## 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端 | Next.js (App Router) + Tailwind CSS + shadcn/ui |
| 后端 | Node.js + Express |
| 数据库 | PostgreSQL + Sequelize ORM |
| 缓存 | Redis |
| 文件存储 | MinIO |
| 日志 | Winston |
| 部署 | Docker + Docker Compose / PM2 |

---

## 核心模块

### 看板（Dashboard）
可视化数据概览，支持通过自定义 SQL 配置图表数据源，无需改动代码即可调整展示内容。适合运营人员日常数据监控。

### 日志（Logging）
基于 Winston 的结构化日志体系，覆盖接口访问日志、操作审计日志、敏感数据访问日志。支持日志分级、按天归档、前端可视化查看。

### 监控（Monitor）
接口级别的健康监控，支持自定义监控规则、告警阈值配置，异常时触发通知（邮件/系统通知），并记录告警历史。

### 系统管理（System）
完整的 RBAC 权限体系，包含：
- 用户管理
- 角色管理（含权限绑定）
- 菜单管理（含权限自动生成）
- 部门管理
- 系统配置
- 数据字典
- 敏感字段配置

---

## 亮点特性

- **代码生成器**：基于数据库表结构一键生成增删改查完整模块
- **数据访问权限**：用户级别的私有数据控制（ALL / 部门 / 本人）
- **数据脱敏**：接口返回自动脱敏敏感字段
- **短信登录**：验证码登录开箱即用
- **接口开发平台**：快速配置系统数据接口，内置鉴权支持
- **第三方对接**：API Key + 签名验证机制，安全支持外部系统接入
- **文件管理**：基于 MinIO 的完整文件管理服务
- **邮件通知**：SMTP 邮件服务，支持模板配置
- **水印服务**：可配置的水印方案
- **登录页定制**：布局与样式可配置

详细说明请查看 [docs/features](./features/) 目录。
